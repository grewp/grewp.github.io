The Moviegoer

Front-End Developer, Founder · 08.15-05.17

The Moviegoer is a student run site I founded, ran, and developed for two years. The site itself consists of two aspects: a user facing site and CMS for use by editors. In Fall of 2016, I sought to modernize both the site and app with the help of a Back-End Engineer.

I worked with my Back-End Engineer to develop a realistic timeline and set of requirements. We planned a two-week sprint followed by two weeks of cleanup; he would take the back-end and dev ops responsibilities, while I took front-end and design responsibilities. Each week leading up to and throughout the sprint we met to make key design decisions. We chose to use isomorphic React to improve site SEO, we integrated AWS Lambda functions into the architecture to optimally resize images, and we made sure the restructured API endpoints were consistent with the new site features and navigation I was designing.

Based on research I had done in the current site’s analytics, I noticed different groups of readers were disproportionately engaging with only a subset of the authors on the site. Reasoning that our target audience were compelled to read articles more by the social clout of the authors rather than the content, I sought out to make the site more personable and use authors as the primary form of site navigation, adding author cards to the side of each article which linked back to a dedicate author page as well as adding a prominent author page containing all of our writers.

I implemented this idea from concept to execution building wireframes and subsequently design mocks for each page. During the design period, I brought on a graphic designer to assist with the development of site typography, specifying not only a system of fonts, but best practice for font sizing, spacing, and coloring to improve readability. I then wrote all of the front-end code in React and CSS with the help of Bourbon Neat’s grid system. The choice of Bourbon Neat over Bootstrap was made because the site’s minimalist design did not necessitate all of the features of Bootstrap. Moreover, since the React architecture was selectively rendering articles that were essentially text nodes (the images were loaded asynchronously after page load) Bootstrap’s large packet size would have slowed down an otherwise snappy technical design. Finally, in the clean up stages of development I ran informal user tests to make sure the site navigation was effective, cross-compatible, and responsive.
